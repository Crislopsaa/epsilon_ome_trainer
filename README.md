## üèóÔ∏è Arquitectura T√©cnica

**EPSILON OME TRAINER** sigue una programaci√≥n modular y basada en eventos de PyQt5, intentando mantener los principios SOLID cuando sea posible. Esto garantiza una experiencia fluida, separando la UI (interfaz) de las interacciones con la base de datos y con la IA.

### 1. El N√∫cleo de la IA (a√∫n a entrenar)

* **Uso de C√≥digo LaTeX:** Tanto el Enunciado como el Enunciado+Soluci√≥n son c√≥digos LaTeX que son convertidos en PDFs y mostrados al usuario (al presentar el problema y al mostrar su soluci√≥n, respectivamente). ### 2. Interfaz y Experiencia de Usuario (PyQt5)

La interfaz utiliza:
* **Visualizaci√≥n Avanzada:** Usa **QWebEngine** para renderizar un entorno web local (**motor Chromium**) dentro de la app de escritorio. Esto se utiliza para lo siguiente:
